# 🍬 Unity 2D Sudoku 

A dynamic, procedurally generated 2D Sudoku puzzle game developed with C# and Unity. 

This project was created to demonstrate the integration of complex matrix algorithms (Backtracking) with a modern, custom UI. Instead of using standard text fonts, the game features a custom "Candy" aesthetic using sliced sprite-based visual buttons for a polished casual gaming experience.

## ✨ Key Features

* **Procedural Level Generation:** Every time the game starts, a completely unique, solvable 9x9 Sudoku grid is generated from scratch in the background.
* **Sprite-Based UI:** Moving away from standard text rendering, the numbers are individually sliced 2D sprite assets (Candy style) mapped to UI Image components.
* **Difficulty Levels:** Players can select from Easy (30 hidden cells), Medium (45 hidden cells), or Hard (55 hidden cells) via the main menu.
* **Real-Time Feedback:** Invalid inputs are instantly detected, changing the cell's visual state to red (error mode). Correct numbers maintain their original color.
* **Win Condition Loop:** The game continuously checks the grid state. Once all cells are correctly filled and match the underlying solution matrix, a "Game Over / Play Again" sequence is triggered.

## 🧠 Algorithms & Architecture

The core architecture relies on performant algorithms to ensure 100% valid puzzles every session:

### 1. Recursive Backtracking
The core engine of the game. It fills an empty 9x9 matrix by adhering to Sudoku rules (unique numbers in rows, columns, and 3x3 sub-grids). When the algorithm hits a dead end, it backtracks to the previous cell and evaluates alternative combinations until a flawless `solutionGrid` is formed.

### 2. Fisher-Yates Array Shuffling
To prevent the backtracking algorithm from generating the exact same layout every time, the array of numbers (1-9) is dynamically shuffled before evaluating each cell. This guarantees infinite randomization (RNG) and replayability.

### 3. Puzzle Subtraction (Masking)
While the perfect solution grid is kept in memory, the playable `puzzleGrid` is generated by randomly selecting coordinates and masking their values (`0`) based on the chosen difficulty level.

### 4. Sprite Slicing & UI Mapping
Numbers are not text elements. They are 2D objects sliced using the Unity Sprite Editor. When a player inputs a value via the Numpad, the script handles the mathematical validation in the background while updating the corresponding Sprite on the Image component in the foreground.

## 🛠️ Technologies Used

* **Game Engine:** Unity 2D Core
* **Language:** C#
* **UI/Visuals:** Unity Canvas, Sprite Editor, UI Layout Groups (Grid, Vertical, Horizontal)
